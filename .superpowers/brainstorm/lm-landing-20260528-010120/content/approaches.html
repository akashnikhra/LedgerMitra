All three use the DESIGN.md color system (cream canvas, ink text, Berkeley Mono).

A

Full Terminal Theme

Follow DESIGN.md exactly. Hero shows a dark TUI mockup — a faux terminal showing accounting commands (sales:list, gst:compute, ledger:balance). ASCII bracket markers for features. Everything monospaced. The landing page reads like a manpage for an accounting CLI tool.

Pros

  • Strongest brand identity
  • Matches existing DESIGN.md perfectly
  • Unique in accounting software space

Cons

  • Less approachable for non-technical users
  • No screenshots of the actual app
B

README / Manpage Style

Hero is a centered README-style block with an ASCII-art wordmark. Clean text layout with section headers using bracketed labels ([+], [*]). No dark TUI mockup — the hero is a large monospaced headline on the cream canvas. More minimal, more approachable, less dramatic.

Pros

  • Cleanest, most minimal design
  • Fastest to implement
  • Approachable for all audiences

Cons

  • Less visually striking
  • Dark hero section is a key brand element in DESIGN.md
C

Terminal + App Screenshots (Recommended)

Terminal-themed hero with dark TUI mockup (matching DESIGN.md). Features section uses actual app screenshots the user already captured — each framed in a thin hairline border with bracketed labels. Brings the credibility of real product screenshots while keeping the terminal aesthetic. Best of both worlds.

Pros

  • Strong brand + product credibility
  • Uses existing screenshots
  • Most effective for driving downloads

Cons

  • Slightly more HTML to build
  • Need to maintain screenshot quality
